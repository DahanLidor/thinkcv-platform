# thinkCV Platform ⚡

פלטפורמת ראייה ממוחשבת לסדנאות והאקתונים (כיתות ז׳–י׳). הילדים לומדים את כל מסלול ה-CV, מאמנים ביחד או לחוד, מגדירים לוגיקות, ומציגים ללקוח (ראש העיר) דשבורד חי עם וידאו מתויג, אנליטיקות ולוג התראות.

> ריפו חדש. buildless (בלי שלב build), מודולרי, ES modules טהורים. עמוד השדרה: Supabase (בהמשך).

## הארכיטקטורה — מנוע אחד, מקורות מתחלפים, מנויים מתחלפים

```
   מקורות                 מנוע                    מנויים
 ┌──────────┐        ┌──────────────┐        ┌──────────────┐
 │ מצלמה    │        │              │        │ דשבורד       │
 │ קובץ     │──frame─▶│  זיהוי → אירוע│──event─▶│ אנליטיקה+לוג │
 │ טלפון    │        │              │        │ ESP32 (בהמשך)│
 └──────────┘        └──────────────┘        │ notebook     │
                     מנוע חוקים על הזרם        └──────────────┘
```

מבנה הזיהוי הקנוני שכל המערכת מסתמכת עליו:
```js
detection = { cls, label, score, x, y, w, h }   // x,y,w,h מנורמלים 0..1
```

```
  dashboard.js        המשטח של הלקוח: וידאו+תיוגים, אנליטיקה חיה, לוג. צרכן טהור (read-only)

  frames.js           חילוץ פריימים חדים מווידאו/תמונות (ניקוד Laplacian)
  annotate.js         כלי תיוג קנבס: ציור/עריכת מלבנים, קטגוריות, קיצורי מקלדת

```

## איך מריצים (שלב 1)

ES modules והמצלמה דורשים הגשה מ-http(server), לא פתיחת קובץ ישירה. `localhost` נחשב מאובטח אז המצלמה תעבוד בלי https:

```bash
cd thinkcv-platform
python3 -m http.server 5173
# פתחו http://localhost:5173  →  לחצו "הפעל מצלמה" → אשרו הרשאה
```

או VS Code → Live Server. אמורים לראות וידאו חי עם מלבנים מצוירים, מדד סיכון, אנליטיקה לפי קטגוריה, ולוג שנדלק כשמזוהה אדם.

בינתיים רץ מודל **COCO גנרי** (80 מחלקות מהמדף) רק כדי להוכיח שכל הצנרת עובדת.

### טעינת מודל Roboflow משלך

בסרגל "מקור המודל" בוחרים "מודל Roboflow שלי" וממלאים שלושה ערכים. **Roboflow לא נותן להוריד קובץ משקולות** — במקום זה, החבילה `inferencejs` טוענת את המודל שלך לדפדפן פעם אחת ואז מריצה אותו מקומית (on-device, מעל TF.js) על כל פריים, בלי לשלוח וידאו לשרת. שלושת הערכים:

- **publishable key** — ב-Roboflow: `Project → Settings` (מתחיל ב-`rf_...`). ציבורי, מיועד לרוץ בדפדפן.
- **שם מודל (slug)** — מזהה הפרויקט (למשל `urban-hazards`).
- **גרסה** — מספר הגרסה המאומנת (למשל `3`).

הכי קל: ב-Roboflow → `Deploy` → דוגמת הקוד ל-inferencejs — שלושת הערכים כבר ממולאים שם, פשוט מעתיקים.

> **הערה על buildless:** `inferencejs` נכתבה לשימוש עם bundler ומריצה worker ברקע. כאן היא נטענת מ-CDN (esm.sh). אם ה-worker לא נטען ככה (סיכון ידוע בסביבה בלי build), **המודול הזה בלבד** (`engine/roboflow.js`) עשוי לדרוש Vite — כל שאר הפלטפורמה נשארת buildless. זה המקום היחיד שהוזהרנו עליו מראש.

### להחליף למודל מקומי (Colab)

אם בעתיד תרצה לטעון `model.json` מקומי (הפלט של Colab), זה `YoloEngine` — מימוש נוסף מאחורי אותו `detect()`, בלי לגעת בשאר.

### לטעון את המודל שאימנת (Colab → yolo_tfjs_model.zip) ✅

בסרגל "מקור המודל" בוחרים "מודל שאימנתי", בוחרים את `yolo_tfjs_model.zip` (הפלט של האימון ב-Colab), ולוחצים "טען מודל". המודל רץ מקומית בדפדפן — אותו פענוח בדיוק כמו שלב "בדיקה" בפלטפורמה הישנה (letterbox + NMS). שמות הקטגוריות נקראים אוטומטית אם הם ארוזים ב-zip; אם לא, אפשר להקליד אותם ידנית (מופרד בפסיקים: `בור, פסולת`), אחרת יופיעו כ-`class 0`, `class 1`.

## מה כבר עובד ומה נבדק

מנוע האירועים, מקור המצלמה, מנוע החוקים, האנליטיקות והדשבורד — כתובים ועברו בדיקת תחביר. מנוע החוקים (לוגיקה טהורה) עבר בדיקות יחידה: ביטחון, כמות, מיקום, AND/OR, מדד סיכון משוקלל, ווקטור-ביטחון פר-קטגוריה (פלט ה-ESP32 העתידי).
